## 1. Stanford University

### CME 241: Foundations of Reinforcement Learning with Applications in Finance
- **Instructor:** Prof. Ashwin Rao | **Term:** Winter 2026
- **Program:** Institute for Computational & Mathematical Engineering (ICME)
- **Topics (3 modules):**
  - Module 1 (3 weeks): Markov Decision Processes, Dynamic Programming, Approximate DP
  - Module 2 (2 weeks): Financial trading as stochastic control -- portfolio management, derivatives pricing/hedging, order-book trading
  - Module 3 (4 weeks): RL algorithms -- Monte Carlo methods, Temporal-Difference, Batch RL, Policy Gradient
  - Final week: Industry guest speaker on practical RL
- **Tools:** Python, Google Colab, GitHub code repository
- **Project:** Personal finance optimization (growing wealth while managing obligations)
- **Textbook:** "Foundations of Reinforcement Learning with Applications in Finance" by Ashwin Rao (free PDF)
- **Website:** https://cme241.github.io/

### STATS 240: Statistical Methods in Finance
- **Units:** 3 | **Prereq:** STATS 200
- **Topics:**
  - Regression analysis and investment models
  - Principal components and multivariate analysis
  - Likelihood inference and Bayesian methods
  - Financial time series
  - Volatility estimation and modeling
  - Statistical methods for portfolio management

### STATS 244: Quantitative Trading -- Algorithms, Data, and Optimization
- **Units:** 2-4 | **Prereq:** STATS 240
- **Topics:**
  - Statistical trading rules and performance evaluation
  - Active portfolio management and dynamic investment strategies
  - Data analytics and models of transactions data
  - Limit order book dynamics in electronic exchanges
  - Algorithmic trading, informatics, and optimal execution
  - Market making and inventory control
  - Risk management and regulatory issues
- **Sequence:** STATS 240 -> STATS 244 forms a natural two-course progression

### Mathematical and Computational Finance Program (MCF)
- **Full interdisciplinary program** preparing students for quant strategist, risk manager, and data scientist roles
- **Topics:** Python & R programming, interest rates, Black-Scholes, financial time series, CAPM, options pricing, optimization methods, ML algorithms

## 2. Carnegie Mellon University (CMU) -- MSCF Program

The Master of Science in Computational Finance is a joint program across four departments: Tepper School of Business, Statistics & Data Science, Mathematical Sciences, and Computer Science. It features a **seven-course data science / ML sequence** (5 core + 2 elective) -- the most extensive among quant finance master's programs.

### Data Science & ML Course Sequence:

#### 46921 Financial Data Science I
- Statistical inference for fitting models to financial data, including complex multivariate forms

#### 46923 Financial Data Science II
- Data exploration and mining, data cleaning, handling modern financial datasets

#### 46926 Machine Learning I
- First in a two-part sequence for quantitative finance
- **Topics:** Foundations of supervised learning, bias-variance tradeoffs, model validation/assessment, classification metrics, regularized regression, nonparametric regression, generalized additive models, support vector machines, tree-based methods

#### 46927 Machine Learning II
- **Topics:** Boosting and ensemble methods, clustering, mixture models and topic modeling, natural language processing, Markov decision processes and reinforcement learning, neural networks and deep learning

#### 46929 Financial Time Series Analysis
- AR, MA, ARMA, ARIMA models in financial applications, forecasting, quantitative trading strategies

#### Deep Learning (elective)
#### Natural Language Processing (elective)

#### 46983 MSCF Machine Learning Capstone Project (Year 2)
- Students collaborate on real-world financial firm challenges using data mining, modeling, visualization, statistical and ML methods

### Other Key MSCF Courses:
- **46944/46945 Stochastic Calculus for Finance I & II** -- arbitrage-free pricing, binomial models, continuous-time markets, derivatives modeling
- **46982 Market Microstructure and Algorithmic Trading** -- trading in equity, options, fixed income, futures, and crypto markets
- **STA 36680 Quantitative Financial Analytics and Algorithmic Trading** -- project-based, data-driven methodologies

## 4. Columbia University

### IEOR E4525: Machine Learning for OR & FE
- **Instructor:** Martin Haugh | **Department:** IE&OR
- **Programs:** MS Financial Engineering, MS Operations Research
- **Detailed Syllabus (16 topics):**
  1. Introduction to Machine Learning
  2. Regression I -- linear regression, bias-variance decomposition
  3. Classification I -- k-NN, Naive Bayes, LDA, QDA, logistic regression, optimal Bayes classifier

  6. Classification II -- trees, bagging, random forests, boosting
  7. Clustering
  8. Introduction to Causality
  9. Support Vector Machines
  10. Kernels and the Kernel Trick (RKHS)
  11. The EM Algorithm
  12. Dimension Reduction -- PCA, kernel PCA, recommender systems, matrix factorization, PageRank
  13. Hidden Markov Models
  14. Bayesian Models and MCMC
  15. Graphical Models -- DAGs, Markov random fields
  16. Variational Inference -- KL divergence, variational Bayes
- **Textbooks:**
  - "An Introduction to Statistical Learning" (James, Witten, Hastie, Tibshirani)
  - "Bayesian Reasoning and Machine Learning" (Barber)
  - "Pattern Recognition and Machine Learning" (Bishop)

## A. Quantitative Researchers at Hedge Funds

| Language | Usage |
|----------|-------|
| **Python** | Primary research language: prototyping, data analysis, ML model development. Pandas was born at AQR Capital Management. |

| **Java** | Used heavily at Two Sigma (with Kubernetes, public cloud); distributed systems |

| Tool | Purpose |

| **XGBoost / LightGBM / CatBoost** | Gradient boosting -- workhorse for tabular financial data |

### Key 2025/2026 Trends

- RL for hedging strategy optimization
- Deep learning reducing pricing errors by 60%+ vs. Black-Scholes for some instruments
- ML-based stress testing frameworks gaining regulatory acceptance

# PART 4: RECOMMENDED LEARNING PROGRESSIONS

---

## Progression 1: Foundational -> Applied (Self-Study Path)

```
Step 1: Python + Math Foundations
  - Python for Data Science (any MOOC)
  - Linear algebra, calculus, probability/statistics

Step 2: Core Machine Learning
  - Stanford CS229 (free lectures) or Coursera ML Specialization
  - scikit-learn, pandas, numpy proficiency

Step 3: Financial Foundations
  - Columbia's Financial Engineering & Risk Management (Coursera)
  - Options pricing, portfolio theory, time series

Step 4: ML for Finance
  - NYU's ML and RL in Finance Specialization (Coursera, 4 courses)
  - Read: "Advances in Financial Machine Learning" (Lopez de Prado)

Step 5: Algorithmic Trading Practice
  - QuantConnect Boot Camp (free)
  - Quantra learning tracks
  - Build and backtest strategies with Zipline/Backtrader

Step 6: Advanced Topics
  - Deep learning for finance (LSTMs, Transformers)
  - Reinforcement learning (Stanford CME 241 materials are free)
  - Alternative data and NLP
```

## Progression 2: University Master's Path

```
Tier 1 (Most Rigorous):
  - CMU MSCF (strongest ML integration, 7-course data science sequence)
  - Princeton MFin + ML Certificate
  - Stanford MCF

Tier 2 (Excellent):
  - Columbia MSFE (IEOR E4525 + broad elective access)
  - NYU Tandon FRE (FRE-GY 7773 + advanced ML)
  - Oxford MSc MCF (strong C++/Python, PyTorch, deep learning)
  - MIT MFin (Sloan + EECS cross-registration)

Free Alternative:
  - WorldQuant University MScFE (accredited, 2-year, entirely free)
```

## Progression 3: Industry Practitioner Upskill

```
Phase 1 (1-2 months): Google Cloud ML for Trading Specialization (Coursera)
Phase 2 (2-3 months): NYU ML & RL in Finance Specialization (Coursera)
```
